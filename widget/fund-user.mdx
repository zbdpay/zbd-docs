## Description

Transfer publisher points to a user's reward account. The user can then withdraw the balance through the widget. Amount is in **currency minor units** — for USD, `100` equals `$1.00`.

This endpoint is idempotent on `idempotency_key`.

## Configuration

### Header Parameters

<ParamField required header="apikey" type="string">
  Your ZBD project API key (production scope).
</ParamField>

<ParamField initialValue="application/json" header="Content-Type" type="string">
  Content Type
</ParamField>

### Body Parameters

<ParamField required body="zbd_user_id" type="string">
  The ZBD user ID returned by Create User.
</ParamField>

<ParamField required body="amount" type="integer">
  Amount in currency minor units. For USD: `100` = $1.00, `1000` = $10.00.
</ParamField>

<ParamField required body="currency" type="string">
  Currency code. Currently only `USD` is supported.
</ParamField>

<ParamField required body="idempotency_key" type="string">
  Unique key to prevent duplicate transfers. Use a UUID.
</ParamField>

<ParamField body="external_id" type="string">
  Optional: your internal transaction identifier for reconciliation.
</ParamField>

<ParamField body="description" type="string">
  Optional: human-readable description shown in transaction history.
</ParamField>

<RequestExample>
